# VedaAI — AI Assessment Creator

AI-powered exam paper generator for teachers. Fill out a form, get a fully structured question paper with sections, difficulty tags, marks, and an answer key — generated by AI, validated before rendering, never raw output on screen.

🔗 **Live Site:** https://veda-ai-platform.vercel.app  
🔗 **Backend API:** https://vedaai-platform.onrender.com

> First request on the live backend may take ~30 seconds to wake up (Render free tier). A notice is shown in the UI during this time.

---

## How it works

When a teacher submits the assignment form, the request doesn't go directly to the AI. It gets queued via BullMQ (backed by Redis), a background worker picks it up, calls OpenAI with a structured prompt, validates the JSON response, stores it in MongoDB, and notifies the frontend over WebSocket. The page updates in real time.

```
Form submit
  → POST /api/assignments
  → Job pushed to BullMQ queue
  → Worker picks up job
  → Calls OpenAI with structured prompt
  → Parses & validates JSON response
  → Saves to MongoDB
  → WebSocket event fires
  → Frontend renders structured paper
```

If MongoDB or Redis are unavailable, the app automatically falls back to an in-memory database and local queue — so it keeps working even without external services.

## Tech Stack

**Frontend** — Next.js 14 (App Router), Tailwind CSS, Zustand, Socket.IO Client, React Hook Form + Zod, `@react-pdf/renderer`, Framer Motion

**Backend** — Node.js + Express (ES Modules), MongoDB with Mongoose, Redis + BullMQ, Socket.IO

**AI** — OpenAI API (`gpt-4o-mini`) with a curriculum-based mock generator fallback if the API is rate-limited or offline

## Getting Started

### Option 1 — Unified monorepo (recommended)

Spin up MongoDB and Redis first:

```bash
docker-compose up -d
```

Then install and run everything from the root:

```bash
npm run install-all
npm run dev
```

Frontend runs at http://localhost:3000  
Backend runs at http://localhost:5000

---

### Option 2 — Manual setup

**Backend**

```bash
cd backend
cp .env.example .env
# Fill in your values (see Environment Variables below)
npm install
npm run dev
```

**Frontend**

```bash
cd frontend
npm install
npm run dev
```

## Environment Variables

Create `backend/.env` with the following:

```env
PORT=5000
MONGODB_URI=mongodb://127.0.0.1:27017/veda-ai
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
OPENAI_API_KEY=your-openai-api-key
```

Create `frontend/.env.local` with:

```env
NEXT_PUBLIC_API_URL=http://localhost:5000
```

## Features

- **Multi-step assignment form** — file upload (drag and drop), due date, subject, grade, topic, question types, marks, difficulty distribution, and additional instructions. All fields validated with Zod before submit.
- **AI question generation** — structured prompt with enforced JSON output. Sections A and B, question types (MCQ, Short, Long, True/False), difficulty tags (Easy / Moderate / Hard), marks per question, and answer key.
- **Real-time updates** — Socket.IO broadcasts job status (`queued → processing → completed / failed`). Zustand store updates cards live without page refresh.
- **Proper exam paper layout** — A4 centered layout with school header, Name/Roll/Section fill lines, section titles, instructions, numbered questions, difficulty badges, and marks tags.
- **PDF export** — client-side A4 PDF using `@react-pdf/renderer`. Answer key on a separate page. Not a browser print screenshot.
- **Collapsible answer key** — expandable solutions panel at the bottom of the output page.
- **Mobile responsive** — sidebar collapses to a top header on mobile, dark bottom tab bar (Home, Assignments, Library, AI Toolkit), single-column card grid, responsive form steps.
- **Offline resilience** — in-memory DB and queue fallbacks if MongoDB or Redis are down.
